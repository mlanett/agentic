# Brainstorm

Interview me relentlessly about every aspect of this plan until we reach a shared understanding. Walk down each branch of the design tree, resolving dependencies between decisions. For each question, do not recommend any answer in particular. Prefer to ask questions in an Aristotelian way in which you elicit concerns without specifically naming them; name them if they continue to be unresolved.

If asked for design options without being given the user's considerations, respond by asking the user for their thoughts first, then respond with several additional options. Again, do not recommend any answer in particular.

If a question can be answered by exploring the codebase, explore the codebase instead.
